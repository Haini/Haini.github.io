Vivado 2018.3 and Locales Settings

Posted on December 17, 2018

If one uses Vivado 2018.3 and encounters the problem of not being able to add new IPs to an existing design, confronted with the following error message (in my example for the clocking wizard):

create_bd_cell -type ip -vlnv xilinx.com:ip:clk_wiz:6.0 clk_wiz_0
FALSE
ERROR: [IP_Flow 19-3188] Error occurred while initializing 'design_1_clk_wiz_0_0'
Tcl error in update procedure while setting value 'MMCM' on the parameter 'PRIMITIVE'. FALSE

ERROR: [IP_Flow 19-3428] Failed to create Customization object design_1_clk_wiz_0_0
CRITICAL WARNING: [IP_Flow 19-973] Failed to create IP instance 'design_1_clk_wiz_0_0'. Error during customization.
ERROR: [BD 41-1712] Create IP failed with errors
ERROR: [BD 5-7] Error: running create_bd_cell  -vlnv xilinx.com:ip:clk_wiz:6.0 -type ip -name clk_wiz_0 .
ERROR: [Common 17-39] 'create_bd_cell' failed due to earlier errors.
endgroup

This problem may be based on the wrong usage of the locales variable, as Vivado is expecting en_US formatting as described in this Xilinx Forum Post. It should be fixed by the following steps

$ export LANG=en_US
$ vivado

In my case, this yielded the following error on start of Vivado:

$ vivado
terminate called after throwing an instance of 'std::runtime_error' 
	what():  locale::facet::_S_create_c_locale name not valid
	/opt/Xilinx/Vivado/2018.3/bin/loader: line 213: 25736 Aborted (core dumped) "$RDI_PROG" "$@"

which in turn can be fixed by narrowing down the locales setting to:

$ export LANG=en_US.UTF-8
