Optional MathJax Integration

Posted on September 25, 2016

In my introduction post I mentioned that it was not optimal to include the MathJax Javascript reference in every header on every page. Indeed those 50kB get pulled down everytime someone opens a page, even if it does not contain any math.

It bothered me and so I changed it. Here is how.

There is a context for that!

In my site.hs I added a new context: 

mathCtx :: Context a
mathCtx = field "mathjax" $ \item -> do 
    mdata <- getMetadata (itemIdentifier item)
    return $ if isJust $ lookupString "mathjax" mdata
        then "<script type=\"text/javascript\" async src=\"https://cdn.mathjax.org/mathjax/latest/MathJax.js?config=TeX-MML-AM_CHTML\"></script>"
        else ""

A context is a monoid, provided by the Hakyll library itself. We can use this context to replace a field with a value. But what is a field? A field is a placeholder with the following syntax: $FieldName$.

So what this context basically does is:

  • Load the metadata of my post
  • Check if the string “mathjax” is in there.
  • If it is in there, return the script string.
  • If not, return nothing.

For the final glory you need to make use of the binary operation of the context class.

I put the script into the header of default.html - and at every place where I use the loadAndApplyTemplate function (??) on the default.html template I need to extend the given context by my mathCtx. This happens by applying the binary operation of the context class between any context and my mathCtx, for example:

>>= loadAndApplyTemplate "templates/default.html" (mathCtx `mappend` postCtx)
